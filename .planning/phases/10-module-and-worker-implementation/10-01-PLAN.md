<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ToolExecutor -> YandexMapsModule | Internal trusted boundary. Input already validated by ToolExecutor (non-blank query/location, clamped max_results). |
| YandexMapsModule -> Child Process Worker | IPC channel. Module sends typed messages; worker runs in isolated Node.js process. No DB access. |
| Child Process -> Yandex Maps (external) | Untrusted external service. Worker receives HTML from Yandex servers. All extracted data treated as untrusted. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Tampering | Worker start message | mitigate | Validate requestId format in worker before processing (UUID check) |
| T-10-02 | Denial of Service | YandexMapsModule timeout | mitigate | 10-minute hard timeout kills worker process; cancelSearch available for user-triggered abort |
| T-10-03 | Information Disclosure | Worker stderr | accept | Worker stderr logged locally only; no PII in worker process itself |
| T-10-04 | Tampering | Yandex Maps HTML response | mitigate | All extracted text trimmed and validated before structuring; no script execution |
</threat_model>
