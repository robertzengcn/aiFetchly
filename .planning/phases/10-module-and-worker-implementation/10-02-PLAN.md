```typescript
function deduplicate(results: YandexMapsBusinessResult[]): YandexMapsBusinessResult[] {
  const seen = new Set<string>();
  return results.filter((r) => {
    const key = r.yandex_id ?? `${r.name}|${r.address}`;
    if (seen.has(key)) return false;
    seen.add(key);
    return true;
  });
}
```

```typescript
async function detectCaptcha(page: Page): Promise<boolean> {
  try {
    const title = await page.title();
    const html = await page.content();
    return html.indexOf("captcha") !== -1 ||
           html.indexOf("robot") !== -1 ||

  }
}
```

7. **scrapeYandexMaps function (main scraping loop):** Mirrors scrapeGoogleMaps with these Yandex-specific differences:

   a. **URL construction:** `const searchUrl = \`https://yandex.ru/maps/?text=${encodeURIComponent(query + " " + location)}\`;`

   c. **Captcha check after navigation:** Immediately after `page.goto()`, call `detectCaptcha(page)`. If true:

   d. **Results container selector:** Instead of Google's `[role="feed"]`, use Yandex-specific selectors. Use multiple fallback strategies per the research:

      - Wait for any of these with `page.waitForSelector()` with 15s timeout.
      - IMPORTANT: These selectors are based on training knowledge and may need adjustment during testing. The code should try multiple selectors and log which one matched.

   e. **Scrolling loop:** Same pattern as Google -- scroll the results container, check for new cards, stop after maxNoNewCards (3) or maxScrollTimeMs (60s). Check `isCancelled` flag each iteration.

   f. **Card extraction loop:** For each result card:
      - Check `isCancelled` at the start of each iteration
      - Click the card to open detail panel
      - Wait for detail panel to appear (try multiple selectors)
      - Call `extractBusinessData(page)` to get structured data
      - Check captcha again after clicking into detail
      - Go back to results (same `goBack` pattern as Google)
      - Send progress updates at each card

   g. **Result normalization:** After collection and deduplication, normalize each result:
      - Trim whitespace from all string fields (name, rating, category, address, phone, website, hours)
      - Ensure consistent field casing (no transformation needed -- fields are already lowercase)

   h. **Summary builder:** Mirror GoogleMapsWorker's `buildSummary()` but output Cyrillic text as-is (no transliteration).

   i. **Final result:** Send `{ type: "result", requestId, success: true, data: result }` with the complete `YandexMapsSearchResult`.

   - `review_count`: Parse review count from text
   - `category`: Try `[class*="category"]`, `[class*="rubric"]`
